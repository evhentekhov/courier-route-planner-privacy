Privacy Policy for Courier Route Planner

Last Updated: January 28, 2026

Introduction

Courier Route Planner ("we", "our", or "the app") is committed to protecting your privacy. This Privacy Policy explains how we collect, use, and share information when you use our mobile application.

Information We Collect

1. Location Data

What we collect:

  • Your current location (latitude and longitude coordinates)
  • Addresses you search for or add to routes
  • Location coordinates for route stops

How we collect it:

  • We request foreground location permission when you use the app
  • Location is accessed only when the app is active (not in background tracking)
  • Location is used to:
    • Set your starting point for routes
    • Optimize delivery routes
    • Display your position on the map
    • Bias address search results to your area
    • Preload map tiles for faster performance

Permission required:

  • iOS: "Location When In Use"
  • Android: ACCESS_FINE_LOCATION and ACCESS_COARSE_LOCATION

2. Route Information

What we collect:

  • Route names and scheduled start times
  • Stop addresses and coordinates
  • Route optimization data (distances, travel times, ETAs)
  • Route progress (which stops are completed)
  • Timestamps for route creation and completion

Where it's stored:

  • Cloud: Firebase Firestore (Google Cloud)
  • Local device: Temporary draft routes and recent searches

3. Device Information

Automatically collected:

  • Device model and manufacturer
  • Operating system version (iOS/Android)
  • App version and build number
  • Platform type (iOS/Android)
  • Device identifier (Android: ANDROID_ID, iOS: IDFV)

Purpose: Analytics and app performance monitoring

4. Usage Data

We collect analytics about:

  • Features you use (optimize route, navigate, etc.)
  • Screens you view
  • App launches and session duration
  • Number of routes created and completed
  • Number of stops optimized
  • Search queries and address selections

Service used: Firebase Analytics (Google)

5. User Preferences

Stored locally on your device:

  • Preferred navigation app (Google Maps, Waze, etc.)
  • Distance unit preference (kilometers or miles)
  • Onboarding completion status
  • App review status
  • Recent address searches (last 5 addresses)

6. Account Information

What we collect:

  • Anonymous user ID (no email or password required)
  • Subscription tier (free or premium)
  • Usage limits and counters (stops per route, monthly limit)
  • Account creation date

Note: We use Firebase Anonymous Authentication. No email, name, or password is collected.

How We Use Your Information

Primary Uses

  1. Route Optimization: Calculate the most efficient order for your delivery stops
  2. Navigation: Display routes on maps and provide turn-by-turn navigation via third-party map apps
  3. Service Improvement: Understand how users interact with the app to improve features
  4. Subscription Management: Track usage limits for free vs. premium users
  5. Local Storage: Save recent searches and preferences for convenience

We DO NOT:

  • Track your location in the background
  • Sell your data to third parties
  • Use your data for advertising
  • Share your data with other users

Third-Party Services

1. Firebase (Google)

Services used: Authentication (anonymous sign-in), Firestore database (route and user data storage), Analytics (usage tracking)

Data shared: User ID, device info, route data, usage events

Privacy policy: https://firebase.google.com/support/privacy

2. Mapbox

Service: Route optimization and directions API

Data shared: Stop coordinates (latitude/longitude) for route calculation

Privacy policy: https://www.mapbox.com/legal/privacy

3. Photon by Komoot

Service: Address geocoding and search

Data shared: Search queries, user location coordinates (for location bias)

Privacy policy: https://www.komoot.com/privacy

4. CartoDB

Service: Map tiles for displaying maps

Data shared: Tile coordinates (derived from visible map area)

Privacy policy: https://carto.com/legal/privacy/

Data Storage and Security

Cloud Storage (Firebase Firestore)

  • Location: Google Cloud servers (location depends on Firebase project region)
  • Security: Encrypted in transit (HTTPS) and at rest
  • Access control: Users can only access their own data (enforced by security rules)

Local Storage

  • Encrypted: User authentication ID (SecureStore)
  • Unencrypted: Route drafts, recent searches, preferences (AsyncStorage)
  • Deletion: Cleared when app is uninstalled

Security Measures

  • All network requests use HTTPS encryption
  • Firebase Security Rules prevent unauthorized access
  • No password storage (anonymous authentication)
  • Data scoped to individual user accounts

Data Retention

  • Routes: Stored for 30 days from the last modification date, then automatically deleted. You can also delete routes manually at any time from the "My Routes" screen.
  • User profile: Retained while your account is active. Automatically deleted after 30 days of inactivity (no app usage).
  • Recent searches: Last 5 addresses only, cleared when app is uninstalled
  • Analytics: Subject to Firebase Analytics retention policies (typically 2-14 months)
  • Local cache: Cleared when you uninstall the app

Note: The 30-day retention period ensures your data is not kept longer than necessary while giving you sufficient time to access recent routes.

Your Rights and Choices

You can:

  1. Delete routes: Delete any route from the "My Routes" screen
  2. Clear recent searches: Automatically limited to last 5 addresses
  3. Change preferences: Update distance units and navigation app in Settings
  4. Delete all data: Uninstall the app to remove all local data
  5. Request data deletion: Contact us to delete your Firebase data

Location Permission:

  • You can revoke location permission in device settings at any time
  • The app will function with limited features without location access
  • No background location tracking - location is only accessed when app is in use

Notifications:

  • You can disable notifications in device settings
  • Notifications are used only for active route navigation reminders
  • No promotional or marketing notifications

Children's Privacy

This app is not directed to children under 13 years of age. We do not knowingly collect personal information from children under 13. If you believe a child has provided us with personal information, please contact us.

Changes to This Policy

We may update this Privacy Policy from time to time. We will notify you of any changes by:

  • Updating the "Last Updated" date at the top of this policy
  • Showing an in-app notice for significant changes (if applicable)

Your continued use of the app after changes indicates acceptance of the updated policy.

Data Deletion Request

To request deletion of your data:

  1. Uninstall the app (deletes all local data)
  2. Email us to request Firebase data deletion (provide your device ID or approximate account creation date)
